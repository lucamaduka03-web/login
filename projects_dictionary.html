Dictionary

Lookup words, definitions and example usage with a clean, fast UI.

Open Live Demo Local Demo

Overview

Simple dictionary app focused on fast lookups, clear definition layouts, and keyboard-first navigation.

Problem

Many dictionary demos are cluttered or slow; users need quick access to concise definitions and examples.

Goals

  • Fast response for lookups
  • Readable, scannable definition layout
  • Mobile-friendly and keyboard accessible

Role

Solo frontend work: UI design, API wiring (client-side), caching and deploy.

Solution

Client-side fetch to a public dictionary API with simple caching and clear error states; includes pronunciation and example sentences when available.

Implementation highlight

// simple cache (localStorage)
async function lookup(term){
  const key = `dict:${term}`;
  const cached = localStorage.getItem(key);
  if(cached) return JSON.parse(cached);
  const res = await fetch(`/api/dictionary?q=${encodeURIComponent(term)}`);
  const json = await res.json();
  localStorage.setItem(key, JSON.stringify(json));
  return json;
}

Accessibility & Performance

  • ARIA roles for live results and alerts
  • Minimal DOM updates and response caching

Results & Next steps

Good baseline for quick lookups; next: offline-first mode and better pronunciation audio support.
